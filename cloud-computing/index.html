Gambaran Umum

Pada project ini, setiap kelompok diminta untuk menjalankan aplikasi web menggunakan Docker Compose dari repository publik yang telah ditentukan, lalu mengeksposnya ke internet menggunakan Ngrok agar dapat diakses oleh siapa saja — termasuk dosen — tanpa perlu server cloud berbayar.

Tidak ada coding dari nol. Fokus sepenuhnya pada deployment, konfigurasi container, networking, dan tunneling.

Penting: Setiap kelompok mendapat repo yang berbeda. URL Ngrok setiap kelompok bersifat unik dan real-time. Tidak ada cara untuk "copy-paste" hasil kelompok lain.

Alur Pengerjaan

1
Clone Repository & Periksa Struktur

Clone repo yang telah ditentukan. Baca README.md dan pahami struktur file sebelum menjalankan apapun.

bash git clone https://github.com/[repo-kelompok]
cd [nama-folder]
# Lihat isi docker-compose.yml
cat docker-compose.yml
ls -la
Catat: berapa service yang ada, port mana yang di-expose, environment variable apa yang bisa dikustomisasi.
2
Terapkan Kustomisasi Wajib

Setiap kelompok wajib melakukan minimal 1 modifikasi pada docker-compose.yml atau file .env sebelum menjalankan container. Modifikasi ini harus didokumentasikan.

Contoh modifikasi yang valid:

  • Mengubah nama database atau kredensial default
  • Mengganti port expose ke host
  • Menambahkan volume untuk persistensi data
  • Menambahkan environment variable baru
3
Jalankan dengan Docker Compose
bash # Jalankan semua service di background
docker compose up -d

# Verifikasi semua container berjalan
docker ps

# Cek log jika ada masalah
docker compose logs -f

Verifikasi aplikasi bisa diakses di http://localhost:[port] dari browser host sebelum lanjut ke langkah berikutnya.

4
Install & Konfigurasi Ngrok
bash — Linux/Ubuntu # Download dan install Ngrok
curl -sSL https://ngrok-agent.s3.amazonaws.com/ngrok.asc \
  | sudo tee /etc/apt/trusted.gpg.d/ngrok.asc > /dev/null
echo "deb https://ngrok-agent.s3.amazonaws.com buster main" \
  | sudo tee /etc/apt/sources.list.d/ngrok.list
sudo apt update && sudo apt install ngrok

# Daftarkan authtoken (dari akun ngrok.com)
ngrok config add-authtoken [YOUR_TOKEN]

# Tunnel ke port aplikasi
ngrok http [PORT]
Daftar akun gratis di ngrok.com untuk mendapatkan authtoken. Akun gratis sudah cukup untuk keperluan ujian ini.
5
Verifikasi & Kirim URL ke Dosen

Setelah Ngrok berjalan, akan muncul URL publik seperti:

  • Kirim URL ke dosen melalui LMS atau group yang ditentukan
  • Screenshot Ngrok terminal + halaman web yang terbuka dari URL publik
  • Buka http://localhost:4040 untuk melihat Ngrok dashboard (request log) — screenshot ini wajib masuk laporan

💡 Hint Umum — Semua Kelompok ▼
H0 Persiapan: pastikan Docker sudah terinstall dengan benar +

Sebelum mulai, verifikasi Docker dan Docker Compose tersedia:

bash # Cek versi Docker
docker --version
# Output yang diharapkan: Docker version 24.x.x atau lebih baru

# Cek Docker Compose (plugin baru, tanpa tanda hubung)
docker compose version
# Output yang diharapkan: Docker Compose version v2.x.x

# Pastikan Docker daemon berjalan
docker ps
# Jika error "Cannot connect to the Docker daemon" → jalankan Docker Desktop atau: sudo systemctl start docker
Catatan Windows: Pastikan Docker Desktop sudah dibuka dan berjalan (icon di system tray). Jika menggunakan WSL2, jalankan semua perintah di dalam terminal WSL, bukan PowerShell.
H1 Container langsung exit setelah docker compose up — kenapa? +

Jalankan docker compose logs [nama-service] untuk melihat pesan error. Penyebab umum:

  • Port sudah dipakai proses lain → cek dengan sudo lsof -i :[port] (Linux/Mac) atau netstat -ano | findstr :[port] (Windows), matikan prosesnya
  • Environment variable tidak lengkap → pastikan file .env ada dan diisi dengan benar
  • Image gagal pull → cek koneksi internet, coba docker pull [nama-image] manual
  • Dependency belum siap → database kadang butuh 10–30 detik sebelum siap menerima koneksi dari app
# Lihat log spesifik satu service
docker compose logs --tail=50 app

# Jalankan tanpa -d untuk lihat output langsung di terminal
docker compose up

# Cek status semua container
docker compose ps

H3 Ngrok error "authentication failed" atau "invalid token" +

Authtoken belum didaftarkan atau salah. Langkah perbaikan:

  1. Login ke dashboard.ngrok.com/get-started/your-authtoken
  2. Klik tombol copy di sebelah token
  3. Jalankan: ngrok config add-authtoken TOKEN_ANDA
  4. Verifikasi: cat ~/.config/ngrok/ngrok.yml — pastikan ada baris authtoken: ...
Jangan share authtoken! Token ini terhubung ke akun pribadi. Setiap anggota kelompok sebaiknya pakai akun masing-masing, tapi satu akun cukup untuk menjalankan tunnel selama ujian.
H4 Ngrok berjalan tapi browser menampilkan halaman kosong, error, atau "502 Bad Gateway" +

Ngrok sudah jalan tapi aplikasi di dalam Docker belum merespons. Cek urutan ini:

  1. Test localhost dulu: buka http://localhost:[port] di browser. Kalau localhost saja gagal, Ngrok tidak akan bisa — selesaikan masalah Docker dulu.
  2. Port tidak cocok: pastikan port yang kamu berikan ke Ngrok sama dengan yang ada di docker ps kolom PORTS.
  3. App butuh waktu: beberapa aplikasi butuh 30–60 detik inisialisasi database. Tunggu dan coba lagi.
  4. Ngrok Interstitial Page: pertama kali buka URL Ngrok di browser akan ada halaman peringatan "This site was tunneled through Ngrok" — klik "Visit Site" untuk lanjut.
# Cek port yang benar-benar di-expose
docker ps --format "table {{.Names}}\t{{.Ports}}"

# Contoh output:
# app_1 0.0.0.0:3000->3000/tcp
# → berarti jalankan: ngrok http 3000
H5 Cara aman mengubah docker-compose.yml tanpa menghapus data +

Perbedaan penting antara down dan down -v:

# AMAN: Stop container, volume data TETAP ada
docker compose down

# Edit file
nano docker-compose.yml

# Jalankan ulang
docker compose up -d

─────────────────────────────────────────
# RESET TOTAL: Stop + hapus volume (data hilang!)
docker compose down -v
# Gunakan hanya jika ingin mulai dari awal bersih
Hati-hati down -v! Flag -v menghapus semua volume Docker yang terdefinisi di compose file. Data database akan hilang permanen. Gunakan hanya kalau memang ingin reset total.
H6 Ngrok berjalan tapi URL berubah terus — bagaimana cara menyimpan URL tetap selama sesi? +

Ini perilaku normal Ngrok versi gratis — setiap kali ngrok http [port] dijalankan ulang, URL berubah. Solusinya:

  • Jangan tutup terminal Ngrok selama sesi berlangsung. Buka terminal baru untuk perintah lain.
  • Jalankan Docker terlebih dahulu, pastikan stabil, baru jalankan Ngrok.
  • Jika Ngrok terpaksa di-restart, segera kirim ulang URL baru ke dosen sebelum presentasi.
Tip presentasi: Buka Ngrok di terminal tersendiri (jangan di tab yang sama dengan Docker logs). Gunakan tmux atau buka beberapa jendela terminal untuk manajemen yang lebih mudah.
H7 Cara membuka dan membaca Ngrok Web Dashboard (localhost:4040) +

Selama Ngrok berjalan, ada web interface lokal yang menampilkan semua request yang masuk melalui tunnel:

  1. Buka browser → ketik http://localhost:4040
  2. Klik tab "Requests" untuk melihat HTTP request yang masuk
  3. Setiap kali seseorang mengakses URL Ngrok kamu, request akan muncul di sini real-time
  4. Klik satu request untuk melihat detail: headers, body, response code
Screenshot wajib! Tampilan dashboard 4040 yang menampilkan minimal 1 request masuk harus ada di laporan. Minta rekan kelompok mengakses URL Ngrok dari HP mereka agar ada request yang muncul.

H11 (Kelompok 6) Cara menambahkan Nginx reverse proxy di depan aplikasi +

Buat file nginx.conf baru, lalu tambahkan service Nginx di compose.yaml:

# File: nginx.conf
server {
  listen 80;
  location / {
    proxy_pass http://app:3000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
  }
}
# Tambahkan di bagian services di compose.yaml:
nginx-proxy:
  image: nginx:alpine
  ports:
    - "8080:80"
  volumes:
    - ./nginx.conf:/etc/nginx/conf.d/default.conf:ro
  depends_on:
    - app

Tunnel Ngrok ke port 8080 (port Nginx), bukan port app langsung. Verifikasi dengan docker ps bahwa ada container nginx-proxy dengan port 8080.

H13 Ada 2 port (8080 dan 8081) — bagaimana cara tunnel keduanya sekaligus? +

Ngrok versi gratis hanya bisa 1 tunnel aktif per sesi. Gunakan 2 terminal secara paralel:

# Terminal 1 — tunnel halaman voting (vote)
ngrok http 8080

# Terminal 2 (buka terminal baru) — tunnel halaman result
ngrok http 8081

Catat kedua URL Ngrok yang muncul. Dalam laporan, sertakan screenshot kedua URL aktif. Untuk membuktikan fungsi: buka URL vote dari HP → pilih opsi → buka URL result dari browser lain → lihat grafik berubah real-time.

Jika error "limit reached": Akun gratis Ngrok membatasi 1 tunnel simultan. Tunjukkan satu per satu: screenshot port 8080 aktif, lalu ganti ke port 8081 untuk screenshot berikutnya. Atau daftar akun Ngrok kedua untuk anggota lain.
H14 Worker container terus restart — status Restarting di docker ps +

Worker bergantung pada Redis dan PostgreSQL yang butuh waktu startup. Ini normal di 1–2 menit pertama. Langkah yang benar:

  1. Tunggu dulu 2 menit setelah docker compose up -d
  2. Jalankan docker compose ps ulang — status Worker harus berubah jadi Up
  3. Jika masih Restarting setelah 5 menit, cek log: docker compose logs worker --tail=30
# Monitor status secara live
watch -n 2 docker compose ps
# Ctrl+C untuk keluar dari watch
H15 (Kelompok 7) Cara scale container vote menjadi 2 replica +

Container yang di-scale tidak boleh punya port host yang fixed (akan konflik). Edit docker-compose.yml terlebih dahulu:

# Di docker-compose.yml, cari bagian service 'vote'
# Ubah dari:
ports:
  - "8080:80"

# Menjadi (expose port tanpa binding host tetap):
ports:
  - "80"  # Docker akan assign port host secara otomatis
# Jalankan dengan scale 2 replica
docker compose up -d --scale vote=2

# Verifikasi ada 2 container vote
docker ps | grep vote
# Harus ada 2 baris dengan nama seperti:
# example-voting-app-vote-1
# example-voting-app-vote-2
Screenshot yang dibutuhkan: Output docker ps yang menampilkan 2 container vote berjalan bersamaan + penjelasan di laporan tentang kegunaan scaling.

  3. App butuh waktu: beberapa aplikasi butuh 30–60 detik inisialisasi database. Tunggu dan coba lagi.
  4. Ngrok Interstitial Page: pertama kali buka URL Ngrok di browser akan ada halaman peringatan "This site was tunneled through Ngrok" — klik "Visit Site" untuk lanjut.

H5 Cara aman mengubah docker-compose.yml tanpa menghapus data +

Perbedaan penting antara down dan down -v:

# AMAN: Stop container, volume data TETAP ada
docker compose down

# Edit file
nano docker-compose.yml

# Jalankan ulang
docker compose up -d

─────────────────────────────────────────
# RESET TOTAL: Stop + hapus volume (data hilang!)
docker compose down -v
# Gunakan hanya jika ingin mulai dari awal bersih
Hati-hati down -v! Flag -v menghapus semua volume Docker yang terdefinisi di compose file. Data database akan hilang permanen. Gunakan hanya kalau memang ingin reset total.
H6 Ngrok berjalan tapi URL berubah terus — bagaimana cara menyimpan URL tetap selama sesi? +

Ini perilaku normal Ngrok versi gratis — setiap kali ngrok http [port] dijalankan ulang, URL berubah. Solusinya:

  • Jangan tutup terminal Ngrok selama sesi berlangsung. Buka terminal baru untuk perintah lain.
  • Jalankan Docker terlebih dahulu, pastikan stabil, baru jalankan Ngrok.
  • Jika Ngrok terpaksa di-restart, segera kirim ulang URL baru ke dosen sebelum presentasi.
Tip presentasi: Buka Ngrok di terminal tersendiri (jangan di tab yang sama dengan Docker logs). Gunakan tmux atau buka beberapa jendela terminal untuk manajemen yang lebih mudah.
H7 Cara membuka dan membaca Ngrok Web Dashboard (localhost:4040) +

  1. Buka browser → ketik http://localhost:4040

  3. Setiap kali seseorang mengakses URL Ngrok kamu, request akan muncul di sini real-time
  4. Klik satu request untuk melihat detail: headers, body, response code

H11 (Kelompok 6) Cara menambahkan Nginx reverse proxy di depan aplikasi +

Buat file nginx.conf baru, lalu tambahkan service Nginx di compose.yaml:

# File: nginx.conf
server {
  listen 80;
  location / {
    proxy_pass http://app:3000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
  }
}
# Tambahkan di bagian services di compose.yaml:
nginx-proxy:
  image: nginx:alpine
  ports:
    - "8080:80"
  volumes:
    - ./nginx.conf:/etc/nginx/conf.d/default.conf:ro
  depends_on:
    - app

Tunnel Ngrok ke port 8080 (port Nginx), bukan port app langsung. Verifikasi dengan docker ps bahwa ada container nginx-proxy dengan port 8080.

H13 Ada 2 port (8080 dan 8081) — bagaimana cara tunnel keduanya sekaligus? +

Ngrok versi gratis hanya bisa 1 tunnel aktif per sesi. Gunakan 2 terminal secara paralel:

# Terminal 1 — tunnel halaman voting (vote)
ngrok http 8080

# Terminal 2 (buka terminal baru) — tunnel halaman result
ngrok http 8081

Catat kedua URL Ngrok yang muncul. Dalam laporan, sertakan screenshot kedua URL aktif. Untuk membuktikan fungsi: buka URL vote dari HP → pilih opsi → buka URL result dari browser lain → lihat grafik berubah real-time.

Jika error "limit reached": Akun gratis Ngrok membatasi 1 tunnel simultan. Tunjukkan satu per satu: screenshot port 8080 aktif, lalu ganti ke port 8081 untuk screenshot berikutnya. Atau daftar akun Ngrok kedua untuk anggota lain.
H14 Worker container terus restart — status Restarting di docker ps +

Worker bergantung pada Redis dan PostgreSQL yang butuh waktu startup. Ini normal di 1–2 menit pertama. Langkah yang benar:

H15 (Kelompok 7) Cara scale container vote menjadi 2 replica +

Container yang di-scale tidak boleh punya port host yang fixed (akan konflik). Edit docker-compose.yml terlebih dahulu:

# Di docker-compose.yml, cari bagian service 'vote'
# Ubah dari:
ports:
  - "8080:80"

# Menjadi (expose port tanpa binding host tetap):
ports:
  - "80"  # Docker akan assign port host secara otomatis
# Jalankan dengan scale 2 replica
docker compose up -d --scale vote=2

# Verifikasi ada 2 container vote
docker ps | grep vote
# Harus ada 2 baris dengan nama seperti:
# example-voting-app-vote-1
# example-voting-app-vote-2

H22 MongoDB tidak terkoneksi — app error "MongoNetworkError" +

Penyebab paling umum: nama host MongoDB di connection string tidak sesuai dengan nama service Docker.

# Cek nama service MongoDB di docker-compose.yml
grep -A 3 "mongo\|db:" docker-compose.yml

# Nama service di sini dipakai sebagai hostname!
# Jika nama servicenya "mongo", maka connection string harus:
MONGO_URI=mongodb://mongo:27017/appdb

# BUKAN: mongodb://localhost:27017/appdb (ini tidak akan bisa)
# BUKAN: mongodb://127.0.0.1:27017/appdb

Edit environment variable di docker-compose.yml bagian service app, sesuaikan MONGO_URI-nya, lalu restart.
